# GameDev Hub

Self-hosted веб-сервис для командной разработки игры на Godot (экспорт в HTML5).
Один контейнер = git-сервер, kanban-доска, запуск игры в браузере и wiki.

Стек: **FastAPI + SQLite** (бэкенд), **React + TypeScript + Vite + Tailwind** (фронтенд).

## Быстрый старт (Docker)

```bash
docker compose up --build -d
```

Сервис доступен в локальной сети: `http://192.168.x.x:8000`
(подставьте IP машины; порт меняется в `docker-compose.yml`).

Все данные (репозиторий игры, база kanban) хранятся в `./data` на хосте.

## Модули

### 0. Репозитории — верхний уровень
Меню репозиториев в сайдбаре: создание, удаление, переключение. Выбранный
репозиторий определяет контекст всего интерфейса: страницы «Игра»,
«Репозиторий» и «Wiki» работают именно с ним. Каждый репозиторий —
самодостаточная единица: своя игра (своя сборка и свой адрес
`/play/<имя>/`), своя wiki в `docs/`, свои ветки. Клонирование:
```bash
git clone http://192.168.x.x:8000/git/<имя>.git
```

### 1. Git
- Вкладка «Ветки»: список с последними коммитами, переключение серверной
  ветки, создание от любой ветки, слияние в текущую (конфликт безопасно
  отменяет операцию), удаление.
- В UI: дерево файлов, просмотр файлов, история коммитов с diff,
  изменения рабочего дерева, commit, переключение и создание веток.
  Хаб — сам по себе origin: разработчики пушат в него, внешних remote нет.
- Репозиторий non-bare с `receive.denyCurrentBranch=updateInstead`:
  push от команды сразу обновляет рабочее дерево, из которого собирается игра.

### 2. Kanban
Колонки Backlog / In Progress / Testing / Done, drag-and-drop,
карточки с описанием, чек-листами и тегами. Хранение — локальный SQLite.

### 3. Игра (главная страница)
Кнопки «Собрать» / «Запустить» / «Перезапустить» / «Во весь экран», лог
сборки в реальном времени, длительность и время последней сборки. После
успешной пересборки игра в iframe перезагружается автоматически. Отдаётся
с заголовками COOP/COEP — Godot 4 Web (SharedArrayBuffer) работает из коробки.

**Автосборка по push**: по умолчанию каждый успешный `git push` сам
запускает пересборку этого репозитория — разработчик пушит, тестировщик на телефоне через
минуту видит новую версию. Отключается галочкой «Собирать при push» на
странице игры (сохраняется в `data/settings.json`).

### 4. Wiki
Markdown-страницы в каталоге `docs/` репозитория. Редактор с живым
предпросмотром, режимы чтение/редактор/сплит, Ctrl+S, шаблоны
(GameDesign, Story, Mechanics, Ideas, Roadmap), опциональный
авто-коммит при сохранении.

## Настройка сборки игры

Кнопка «Собрать» выполняет `BUILD_CMD` в корне репозитория и ожидает
HTML5-билд в `BUILD_OUTPUT` (по умолчанию `build/web`, должен появиться
`index.html`).

Положите в корень репозитория `build.sh` (пример — `examples/build.sh`):

```bash
godot --headless --export-release "Web" build/web/index.html
```

### Сборка Godot в Docker
Базовый образ не содержит Godot. Варианты:
1. Добавить в `Dockerfile` скачивание headless-Godot и export templates.
2. Собирать на хосте/CI и коммитить `build/web` в репозиторий —
   тогда `BUILD_CMD: "true"`, а кнопка «Собрать» просто подтянет билд после pull.

## Переменные окружения

| Переменная       | По умолчанию      | Описание                                   |
|------------------|-------------------|--------------------------------------------|
| `DATA_DIR`       | `./data`          | Каталог данных (репозиторий, SQLite)       |
| `REPOS_DIR`      | `$DATA_DIR/repos` | Каталог всех репозиториев                  |
| `DB_PATH`        | `$DATA_DIR/hub.db`| База kanban                                |
| `BUILD_CMD`      | `bash build.sh`   | Команда сборки (в корне репозитория)       |
| `BUILD_OUTPUT`   | `build/web`       | Каталог HTML5-билда (отн. репозитория)     |
| `DOCS_DIR`       | `docs`            | Каталог wiki внутри репозитория            |
| `FRONTEND_DIST`  | `frontend/dist`   | Собранный фронтенд                         |

## Разработка без Docker

```bash
# бэкенд
cd backend
pip install -r requirements.txt
uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload

# фронтенд (отдельный терминал; dev-прокси на :8000 уже настроен)
cd frontend
npm install
npm run dev   # http://localhost:5173
```

Продакшн-сборка фронтенда: `npm run build` → `frontend/dist`,
FastAPI раздаёт её сам.

## Структура

```
backend/app/
  main.py            # приложение, COOP/COEP, статика, SPA-fallback
  config.py, db.py
  modules/
    gitmod/          # git-операции + Smart HTTP сервер
    kanban/          # задачи (SQLite)
    build/           # менеджер сборки, инкрементальный лог
    wiki/            # markdown-страницы в docs/
frontend/src/
  modules/git|kanban|play|wiki
  components/ui.tsx  # кнопки, модалки, бейджи
  lib/               # api, тема, форматирование
```

Модули независимы: у каждого свой router на бэкенде и свой каталог на фронтенде.
